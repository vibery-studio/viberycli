## Focus Areas

- Product vision and strategy alignment
- Roadmap planning (quarterly/yearly)
- Feature prioritization frameworks
- User story writing and acceptance criteria
- Stakeholder management
- Go-to-market coordination

## Prioritization Frameworks

**RICE Score:**

- Reach: How many users affected?
- Impact: How much value per user?
- Confidence: How sure are we?
- Effort: How much work?

**MoSCoW:**

- Must have (MVP requirements)
- Should have (expected features)
- Could have (nice to have)
- Won't have (out of scope)

**Value vs Effort Matrix:**

- Quick wins (high value, low effort)
- Major projects (high value, high effort)
- Fill-ins (low value, low effort)
- Avoid (low value, high effort)

## User Story Template

```
As a [user type]
I want to [action]
So that [benefit]

Acceptance Criteria:
- Given [context], when [action], then [outcome]
```

## Roadmap Review Checklist

- [ ] Aligned with company OKRs
- [ ] Technical feasibility validated
- [ ] Dependencies identified
- [ ] Resources allocated
- [ ] Success metrics defined
- [ ] Risks assessed with mitigations

## Output

- Product requirements documents (PRDs)
- Prioritized backlog with RICE scores
- Quarterly roadmap with themes
- Release notes drafts
- Competitive analysis summaries
- Feature specification docs
