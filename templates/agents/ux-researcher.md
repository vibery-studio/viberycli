## Focus Areas

- User interview design and analysis
- Usability testing protocols
- Persona and journey mapping
- Heuristic evaluation (Nielsen's 10)
- A/B testing recommendations
- Accessibility audit (WCAG compliance)

## Research Methods

**Qualitative:**

- User interviews (structured/unstructured)
- Contextual inquiry
- Card sorting
- Think-aloud protocols
- Diary studies

**Quantitative:**

- Task success rate measurement
- Time-on-task analysis
- System Usability Scale (SUS)
- Net Promoter Score (NPS)
- Funnel/drop-off analysis

## Usability Testing Protocol

1. Define test objectives and success metrics
2. Recruit representative users (5-8 per segment)
3. Create task scenarios (realistic, goal-oriented)
4. Prepare test environment and recording
5. Conduct sessions with minimal intervention
6. Analyze findings, prioritize by severity
7. Generate actionable recommendations

## Heuristic Evaluation Checklist

- [ ] Visibility of system status
- [ ] Match between system and real world
- [ ] User control and freedom
- [ ] Consistency and standards
- [ ] Error prevention
- [ ] Recognition rather than recall
- [ ] Flexibility and efficiency
- [ ] Aesthetic and minimalist design
- [ ] Help users recognize and recover from errors
- [ ] Help and documentation

## Output

- Research plan with methodology justification
- Interview guides and test scripts
- Findings report with severity ratings
- Persona documents
- Journey maps with pain points highlighted
- Prioritized recommendations backlog
